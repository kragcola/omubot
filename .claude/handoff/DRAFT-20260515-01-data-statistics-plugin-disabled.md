## Summary

设计一个未来的“数据统计”插件，用于基于本地归档数据做可配置群聊统计。插件后续应从 ConversationArchive / MessageLog 读取本地原始记录，只在显式开启“原始数据永不删除”的群聊中提供统计功能。

核心使用期望：

- `/统计 某个词`：统计该词出现次数、使用人、时间分布，并生成饼图或柱状图发到群里。
- 每日统计：按群生成当日长图，包括消息量、文件/链接/表情/小程序统计、话痨排行、当日金句、娱乐向“性压抑指数”排行。
- 默认不启用，不创建运行时插件目录，不写配置，不注册命令。

## Key Design

- 插件名：`data_statistics`，中文显示“数据统计”。
- 存储位置：`storage/plugins/data_statistics/stats.db`。
- 图像输出：复用现有 Pillow，不新增 matplotlib / pandas / pyecharts。
- 数据来源：
  - 文本词频：优先读 `ConversationArchive.conversation_messages.content_text`。
  - 消息类型统计：运行期从 OneBot segment 派生本地特征表；历史数据只能做有限补算。
  - 不重新拉远端历史，不调用 NapCat 补旧图。
- 群资格：
  - 只允许 `chat_type="group"`。
  - 必须显式存在 retention policy，且 `keep_raw_forever=1`。
  - 不满足时 `/统计` 返回：“该群未开启原始记录永久保留，统计不可用。”
- 静默规则：
  - 被动采集可 `silent_safe=True`，但只能写本地派生统计，不能发言、不能设 trigger、不能调 LLM。
  - `/统计` 和每日统计只在 active 群发送。

## Functional Review

### 资格门禁

- 统计功能只面向群聊，不支持私聊、跨群、全局榜单。
- 每次命令和每日任务触发前都必须检查：
  - `chat_type == "group"`。
  - 群配置 `presence_mode == "active"` 才允许发送结果。
  - `conversation_retention_policies` 中存在该群策略。
  - 该策略 `keep_raw_forever=1`，且没有被标记为清理原始记录。
- `silent_learn` 群后续若允许被动采集，也只能更新本地派生统计，不得响应 `/统计`，不得发送日报。
- `off` 群不得采集、不得统计、不得响应、不得发送。
- 不满足资格时，回复固定拒绝文案，不暴露内部表名或配置细节：
  - `该群未开启原始记录永久保留，统计不可用。`
  - `该群当前不是活跃参与模式，统计不可用。`

### `/统计 <词>` 功能审查

- 输入：
  - `<词>` 必填，去除首尾空白后长度至少 1。
  - 默认统计本群，不接受指定其他群。
  - `--days N` 只允许正整数，并设置上限，例如 30 天，避免误触全库扫描。
  - `--bar` 切换柱状图；默认饼图。
- 匹配：
  - 首期只做字面包含匹配，不做正则、不做模糊搜索、不做 LLM 语义扩展。
  - 只统计 `conversation_messages.content_text` 中的文本；图片 OCR、语音转写、文件内容不纳入。
  - 命令消息本身默认不计入，避免刷命令污染结果。
- 输出：
  - 文本摘要：总出现次数、涉及用户数、Top 使用人、最近出现时间、统计时间范围。
  - 图片摘要：饼图显示 Top 使用人占比；柱状图显示按天或按用户分布。
  - 当命中为 0 时只返回简短文本，不生成空图。
- 防滥用：
  - 同一群同一用户需要冷却，例如 30 秒。
  - 单次扫描需要数量上限和超时保护；超过上限时返回“范围过大，请缩短天数”。
  - 不允许通过 `/统计` 查询明显身份敏感、隐私定位、手机号、身份证号等模式；首期可用简单正则拒绝。

### 每日统计功能审查

- 默认关闭，必须管理员显式开启。
- 每日只在 `presence_mode == "active"` 且 `keep_raw_forever=1` 的群发送。
- 发送时间按群配置；缺省值必须保守，例如不自动设置。
- 内容模块分级：
  - 必选：总消息数、活跃人数、人均消息数、文本/图片/表情/文件/链接/小程序数量。
  - 可选：话痨排行、当日金句。
  - 高风险默认关闭：“性压抑指数”。
- 当日金句：
  - 首期只用启发式候选，如长度、互动回复数、非命令、非纯表情。
  - 不调用 LLM；如未来接入 LLM，必须单独开关、单独费用观测、单独超时保护。
- “性压抑指数”：
  - 只能统计显式关键词，不得推断心理或真实性状态。
  - 默认关闭；开启需要管理员二次确认。
  - 推荐默认只展示群级总量或匿名排名；若展示个人排名，必须开启匿名化并标注“娱乐词频指数，不代表真实状态”。

### 数据与存储边界

- 原始聊天记录仍归 ConversationArchive / MessageLog 管理，数据统计插件不得复制完整原文。
- `stats.db` 只保存派生统计、每日快照、配置、审计记录和必要 message 引用。
- 词频查询优先直接读 `conversation_messages.content_text`，但高频日报应使用派生日表，避免反复扫原始表。
- 消息类型统计应在运行期从 OneBot segment 派生，历史数据只做有限补算：
  - 有 `content_json` 的历史消息可以补类型。
  - 没有结构化 segment 的旧消息不做远端补拉。
  - 不调用 NapCat 拉旧图、旧文件或旧小程序详情。
- 图像输出只使用 Pillow 绘制，不新增重依赖；生成文件应放临时目录或插件 storage 下，并设置清理策略。

### 后续最小表结构建议

首期真实实现时可以考虑以下派生表，表名仅为建议，不得在本轮创建：

- `stats_group_settings`：群级开关、发送时间、图表类型、匿名化、娱乐指标开关。
- `stats_message_features`：按消息记录派生类型特征，如 has_text、image_count、sticker_count、file_count、link_count、miniapp_count。
- `stats_daily_snapshots`：按群按日缓存日报所需聚合值。
- `stats_query_audit`：记录 `/统计` 查询词、操作者、范围、是否成功、拒绝原因。

这些表不得保存完整聊天原文；如需要回溯证据，只保存 `conversation_messages.message_pk` 引用。

### Admin 与配置审查

- 首期可以只用群内命令配置，不强制做 Admin 页面。
- 如果后续做 Admin 页面，应嵌入现有群设置或插件详情页，不新增“大而全统计中心”。
- 所有高风险开关必须可见：
  - 是否启用日报。
  - 是否匿名化排行。
  - 是否启用娱乐敏感词频模块。
  - 是否允许 silent_learn 群被动采集派生特征。
- 开关变更必须写审计记录，至少包含 actor、group_id、old_value、new_value、created_at。

## Human-Centered Feature Design

### 开启前的人性化确认

- 管理员执行 `/统计日报 on` 时，不应直接开启完整日报。后续实现应先返回一段确认说明：
  - 会统计哪些内容。
  - 数据来自本群本地归档。
  - 是否展示成员排行。
  - 是否匿名化。
  - “性压抑指数”等娱乐敏感模块默认关闭。
- 建议使用二段式确认：
  - `/统计日报 on` 返回预览和风险说明。
  - `/统计日报 confirm` 才真正开启。
- 首次开启后，第一天只发送“轻量日报”，不含娱乐排行；管理员确认效果后再打开更多模块。

### 友好的命令反馈

- 命令不可用时，不要报内部错误或配置名。使用可理解的话术：
  - `这个群还没开启原始记录永久保留，所以我不能做追溯统计。`
  - `这个群现在是静默学习模式，我只会学习，不会发统计结果。`
  - `这个词今天还没出现过，我先不画空图啦。`
  - `范围有点大，先试试 /统计 词 --days 7。`
- 查询耗时超过 2 秒时，可以先发一个短提示：
  - `我在翻本群本地记录，稍等一下。`
  - 但只允许 active 群发送，且不能频繁刷屏。
- 图片生成失败时，必须退回文本摘要，不让用户看到一串异常。

### 低数据量与冷场处理

- 当日消息数过少时，日报不应硬凑排行榜：
  - 少于 30 条消息：只发轻量摘要，不发排行。
  - 活跃人数少于 5 人：默认隐藏个人排行，避免点名尴尬。
  - 命中人数少于 3 人：饼图合并为“少量样本”，不做个人占比图。
- 如果一天没有足够内容生成“当日金句”，显示：
  - `今天还没有特别适合摘出来的句子。`
- 如果统计维度未知较多，日报必须显示“未知/无法识别”而不是把未知算作 0。

### 排行榜的人性化表达

- “话痨排行”可以在内部仍叫 talkative ranking，但对外文案建议更柔和：
  - `今日活跃发言`
  - `今日聊天贡献`
  - `今天比较能聊`
- 排行默认只显示 Top 5，避免长榜单造成压力。
- 支持管理员配置：
  - `anonymous on`：匿名昵称，如 `成员 A / 成员 B`。
  - `leaderboard off`：关闭个人排行，只展示群级统计。
  - `min_messages_for_ranking`：达到最低消息量才展示排行。
- 不建议展示“倒数排行”“谁最沉默”“谁最性压抑”这类指向性羞辱榜。

### `/统计 <词>` 的可解释体验

- 统计结果应解释匹配方式：
  - `按字面包含统计，不含图片、语音、文件内容。`
  - `统计范围：最近 7 天。`
- 对容易误解的词给出提示：
  - 单字词：提示“单字容易误命中，结果仅供参考”。
  - 常见虚词：提示“这个词太常见，建议换更具体的词”。
  - 正则样式输入：按普通文本处理或拒绝，不执行正则。
- 支持一个轻量 follow-up 体验，但不强制首期实现：
  - 用户刚查完 `/统计 猫` 后，短时间内发 `/统计 --bar` 可复用上一个词。
  - 该上下文只在本群本用户生效，最多保留 2 分钟。

### 每日长图体验

- 长图应偏“群内日报”，不要像监控报表一样冰冷：
  - 顶部：日期、群名、统计时间范围。
  - 第一屏：总消息数、活跃人数、人均消息数、最热时段。
  - 中段：消息类型构成、Top 活跃时段、表情/链接/文件数量。
  - 末段：可选排行、当日金句、娱乐模块。
  - 页脚：`数据来自本群本地归档，娱乐统计仅供参考。`
- 图表颜色应克制，不做刺眼红榜/黑榜。
- 个人昵称需要截断和脱敏，避免过长昵称撑破长图。
- 图片发送失败时发送纯文本日报摘要。

### 金句与娱乐模块边界

- 当日金句不应选取：
  - 明显隐私内容。
  - 手机号、地址、身份证、账号密码等敏感文本。
  - 管理命令、报错堆栈、纯链接。
  - 涉及人身攻击、羞辱或强烈争议的句子。
- 娱乐模块必须具备独立开关，默认关闭。
- “性压抑指数”如果未来实现，建议采用群级趣味标签优先，不默认展示个人排名：
  - 例如 `今日相关词频：12 次，较昨日 +3`。
  - 若管理员确实开启个人匿名排行，只显示匿名 Top N。

## Edge Case Handling

| 场景 | 期望行为 |
| --- | --- |
| 群没有 retention policy | 拒绝统计，提示未开启原始记录永久保留 |
| `keep_raw_forever=0` | 拒绝统计，不做追溯 |
| `presence_mode=silent_learn` | 不响应命令，不发送日报；可选被动派生统计但不得发言 |
| `presence_mode=off` | 不采集、不统计、不发送 |
| 私聊执行 `/统计` | 拒绝，提示该功能只支持群聊 |
| 命令缺少词 | 返回帮助和示例，不扫描数据库 |
| 查询词太短或太泛 | 返回提示，允许用户换更具体的词 |
| 查询词像手机号/身份证/地址 | 拒绝，避免隐私定位 |
| `--days` 非数字或过大 | 返回参数提示，不执行查询 |
| 统计命中为 0 | 返回文本，不生成空图 |
| 命中消息很多 | 使用分页/聚合结果，不回传原文列表 |
| 群成员改名 | 以当时可用 display_name 展示；审计中保存 user_id 哈希或内部引用 |
| 多人同昵称 | 图表展示昵称加短标识，避免混淆 |
| bot 自己发的消息 | 默认排除，避免自我污染；可配置是否纳入 |
| 图片/表情类型无法识别 | 归入未知，不当作 0 |
| 长图生成失败 | 发送文本摘要并记录错误 |
| 数据库忙或超时 | 返回“数据量较大，稍后再试”，不阻塞主循环 |
| 日报发送失败 | 记录失败原因，不无限重试 |
| 群从 active 改 silent/off | 自动停止日报发送 |
| 管理员关闭日报 | 立即停止后续定时任务，保留审计记录 |
| 删除或清理原始记录策略开启 | 自动禁用追溯统计或进入只读历史快照模式 |

## Privacy and Safety Rules

- 不提供“查某个人说过什么”的命令。
- 不提供跨群搜索、跨群排行、全局用户画像。
- 不展示完整原文命中列表；如未来需要证据，只展示少量脱敏片段并要求管理员权限。
- 不统计或输出疑似手机号、身份证号、家庭住址、账号密码、精确定位等敏感模式。
- 不根据词频推断真实心理状态、性状态、政治倾向、健康状态、身份属性。
- 不把娱乐指标接入记忆、黑话、表达方式或 soul，不让临时统计影响 bot 对人的长期判断。
- 不在 silent_learn 群发送“我看到了什么”的提示，避免破坏静默承诺。

## Command Grammar and Aliases

后续真实实现时，命令解析应宽容但不暧昧：支持中文、英文和简写；只响应明确 slash 命令，不从普通聊天自然语言中猜测统计意图。

### 触发词

允许的主触发词：

- 中文：`/统计`、`/数据统计`、`/群统计`、`/查词`。
- 英文：`/stats`、`/stat`、`/data-stats`。
- 简写：`/st`、`/tj`。
- 全角兼容：允许 `／统计`、`／stats`，解析时统一折叠为 `/`。

日报触发词：

- 中文：`/统计日报`、`/群日报`、`/日报`。
- 英文：`/stats daily`、`/daily stats`、`/daily-report`。
- 简写：`/tjrb`、`/dr`。

不支持无 slash 的自然语言触发，例如“统计一下猫”，避免日常聊天误触。

### 参数通用规则

- 英文参数大小写不敏感：`--DAYS` 等同 `--days`。
- 中文参数可用空格或中文冒号：
  - `/统计 猫 天数 7`
  - `/统计 猫 天数：7`
- 英文参数支持空格和等号：
  - `/stats cat --days 7`
  - `/stats cat --days=7`
- 支持短参数：
  - `-d 7` 等同 `--days 7`
  - `-b` 等同 `--bar`
  - `-p` 等同 `--pie`
  - `-a` 等同 `--anonymous`
- 中文图表参数：
  - `柱状图`、`条形图` 等同 `--bar`
  - `饼图`、`占比图` 等同 `--pie`
  - `折线图` 首期可作为趋势图别名，但只有实现趋势功能后才启用
- 多词查询必须支持引号：
  - `/统计 "good morning"`
  - `/stats "good morning" --days 7`
- 未加引号时，主命令后的第一个非参数片段作为查询词；其余片段按参数解析。无法解析时返回帮助，不猜测。

### `/统计` 查询族

基础写法：

- `/统计 <词>`
- `/统计 <词> --days 7`
- `/统计 <词> --bar`
- `/stats <word> -d 7 -b`
- `/st <word> 7d`
- `/tj <词> 最近7天`

时间范围别名：

- `今天` / `today`：当天 00:00 到当前。
- `昨天` / `yesterday`：昨日自然日。
- `本周` / `week`：本周。
- `本月` / `month`：本月。
- `最近7天` / `7d` / `--days 7`：滚动 N 天。
- `--from YYYY-MM-DD --to YYYY-MM-DD`：明确日期范围；首期可延后，但语法预留。

图表与输出：

- `--pie` / `饼图`：按用户占比展示。
- `--bar` / `柱状图`：按用户或日期展示。
- `--trend` / `趋势`：按天趋势图；首期可不实现，返回“趋势图还没开启”。
- `--text` / `文字`：只返回文本摘要，不生成图。
- `--top N` / `前N名`：限制排行数量，默认 Top 5，上限 10。

示例：

```text
/统计 猫
/统计 猫 最近7天 柱状图
/统计 "good morning" --days=14 --bar
/stats cat -d 7 --pie
/st pjsk 30d -b
/tj 猫 今天 文字
```

### 多功能统计命令

除查词外，后续可以逐步开放以下只读功能。所有功能仍受资格门禁、冷却和隐私规则约束。

- `/统计 总览` / `/stats overview`：本群今日轻量总览，文本优先。
- `/统计 今日` / `/stats today`：生成当日截至当前的临时日报预览，不改变日报开关。
- `/统计 昨日` / `/stats yesterday`：重发昨日摘要；每日最多重发一次，避免刷屏。
- `/统计 类型` / `/stats types`：统计文本、图片/表情、文件、链接、小程序数量。
- `/统计 活跃` / `/stats active`：展示活跃时段和活跃人数，不默认点名个人。
- `/统计 词云` / `/stats words`：高频词摘要；首期可只返回 Top 词文本，不生成词云图。
- `/统计 表情` / `/stats stickers`：表情使用概况，只展示群级数量和 Top 表情，不分析个人偏好。
- `/统计 链接` / `/stats links`：链接数量和域名分布，不展示完整 URL。
- `/统计 文件` / `/stats files`：文件数量和类型分布，不展示文件名全文。
- `/统计 金句` / `/stats quote`：当日金句预览，受金句过滤规则约束。
- `/统计 对比 猫 狗` / `/stats compare cat dog`：比较两个词出现次数；首期可延后。
- `/统计 趋势 猫` / `/stats trend cat`：词频趋势；首期可延后。

未实现的多功能命令必须返回明确提示，例如：

```text
这个统计子功能还没开启。当前可用：/统计 <词>、/统计 帮助。
```

### 日报配置命令

管理员命令：

- `/统计日报 on` / `/daily-report on` / `/dr on`：进入开启确认，不直接开启。
- `/统计日报 confirm` / `/dr confirm`：确认开启。
- `/统计日报 off` / `/dr off`：关闭。
- `/统计日报 status` / `/dr status`：查看当前配置。
- `/统计日报 preview` / `/dr preview`：生成一次预览；不改变定时任务。
- `/统计日报 time 23:30` / `/dr time 23:30`：设置发送时间。
- `/统计日报 chart pie|bar` / `/dr chart pie|bar`：设置默认图表。
- `/统计日报 anonymous on|off` / `/dr anon on|off`：匿名化。
- `/统计日报 leaderboard on|off` / `/dr rank on|off`：个人排行。
- `/统计日报 quote on|off` / `/dr quote on|off`：当日金句。
- `/统计日报 fun-index on|off` / `/dr fun on|off`：娱乐敏感词频模块，开启时必须二次确认。

普通成员可用：

- `/统计日报 status`：只展示是否开启、发送时间、匿名化状态和模块开关，不展示审计细节。
- `/统计 帮助`：查看帮助。

### 权限与确认

- 普通成员可执行只读查询，但受群级开关和冷却限制。
- 只有管理员可修改日报配置、匿名化、排行、娱乐模块。
- 高风险开关必须二次确认：
  - 日报首次开启。
  - 关闭匿名化。
  - 开启个人排行。
  - 开启 `fun-index`。
- 二次确认上下文只在本群、本管理员、生效 2 分钟，过期需要重新发起。

### 歧义与错误处理

- 查询词与子命令冲突时，优先按子命令解析。例如 `/统计 今日` 是今日总览，不是统计“今日”这个词。
- 若用户确实要查子命令同名词，使用引号：`/统计 "今日"`。
- 同时给出互斥图表参数时，后出现者优先，并在文本中提示采用了哪个。
- 参数无法解析时不执行查询，只返回一条简短帮助。
- 任何命令都不得把原始异常、SQL、路径或表名发到群里。

## Risk Self-Audit

- 隐私与群同意风险：词频、话痨排行、使用人统计会把普通聊天行为变成可视化画像。后续实现必须只在群级显式开启后生效，并在命令反馈中清楚说明数据来自本群本地归档。
- 留存策略风险：统计依赖原始消息长期保存。任何没有 `keep_raw_forever=1` 的群都不得启用追溯统计，避免功能反向推动不透明的数据保留。
- 静默模式边界风险：silent_learn 群最多只能写本地派生统计，不允许主动发送日报、触发 LLM、设置唤醒条件或注册命令响应；`presence_mode=off` 群必须完全不采集。
- 娱乐指标误读风险：“性压抑指数”只能作为显式关键词娱乐计数，默认关闭，不得推断真实心理、性状态、身份倾向或健康状态；长图中必须标注“娱乐词频指数”。
- 普通图片/文件误采集风险：历史消息中的文件、图片、表情、小程序统计只能记录类型和必要计数，不能为了补全统计主动重新拉远端历史或下载过期资源。
- 性能风险：千人大群的按词扫描和日报生成可能压住 SQLite 和图片渲染。后续实现必须使用 ConversationArchive 游标或按日派生表，不应每次命令全库扫描。
- 误启用风险：本文件位于 `.claude/handoff/`，不是插件目录。任何代理不得据此创建 `plugins/data_statistics/`、写配置、注册 Admin API 或加入启动流程。
- 审计风险：后续真实实现必须为日报开关、匿名化开关、敏感娱乐指标开关写入可追溯 revision 或配置变更记录，避免“谁打开的”不可查。
- 公平性与社交风险：话痨排行和娱乐排行可能造成点名压力。后续实现应支持匿名化、Top N 限制和关闭排行模块。
- 数据正确性风险：历史数据中的图片/表情/文件类型不一定完整，日报必须区分“可识别数量”和“未知数量”，不能把未知当作 0。
- 成本风险：当日金句若未来引入 LLM，可能变成每日固定成本来源。首期不得调用 LLM，后续必须有费用统计和开关。
