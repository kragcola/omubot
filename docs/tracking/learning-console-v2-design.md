### 各阶段 tab 行为差异

| 阶段 | 行内操作 | 排序默认 | 特殊列 |
|------|----------|----------|--------|
| 候选 | 通过 / 拒绝 | 最新优先 | 出现次数 |
| 待审 | 通过 / 退回 | 最新优先 | 审核备注 |
| 已生效 | 归档 / 编辑 | 生效时间 | 使用次数 |
| 命中 | 查看上下文 | 命中次数↓ | 最近命中时间 |
| 归档 | 恢复 / 删除 | 归档时间 | 归档原因 |

### 核心改进（vs 旧页面）

1. **按时间段聚合**（今天/昨天/本周/更早）——同批次提取的词条自然归在一起，不再零散
2. **紧凑行布局**——每条词条一行，关键信息水平排列，不用卡片堆叠
3. **行内操作**——approve/reject 直接在行尾，不用打开抽屉就能快速处理
4. **批量操作**——勾选多条后一键通过/拒绝/归档
5. **阶段 tab 带计数徽标**——切换成本低，当前阶段一目了然

### 数据来源

- 列表 → 现有 `GET /api/admin/learning/items?stage=xxx&cursor=xxx`
- 操作 → 现有各 noun 的 approve/reject/archive API

## 视图 3：设置（Settings）

左侧分组导航 + 右侧表单。顶部为 AI 自动托管主开关，下方为分项设置。

### AI 自动托管（Autopilot）

设置页顶部，优先级最高的控制区：

```
┌─────────────────────────────────────────────────────────┐
│ AI 自动托管                                    [开关]    │
│ 开启后，管线全流程由 AI 自主处理：                        │
│ · 自动提取所有 noun 的候选词条                           │
│ · 候选词条自动进入 AI 审核（不等时间槽）                  │
│ · 审核通过自动生效，拒绝自动归档                         │
│ · 漂移超龄自动归档                                      │
│ · 表达/记忆整合按默认间隔自动运行                        │
│                                                         │
│ 激进程度: [保守 ○ ● ○ 激进]                             │
│ · 保守：仅自动提取，审核仍需人工确认                      │
│ · 标准：自动提取+审核，低置信度暂留待审                   │
│ · 激进：全自动，AI 置信度 >0.6 即通过                    │
└─────────────────────────────────────────────────────────┘
```

**行为规则：**

- 开启后，下方所有分项开关变为灰色只读，显示"由自动托管控制"
- 关闭则回到手动逐项配置
- 激进程度影响 AI 审核的置信度阈值和自动操作范围

**后端实现：**

`learning_settings.json` 顶层新增：

```json
{
  "autopilot": {
    "enabled": false,
    "aggressiveness": "standard"
  },
  "style": { ... },
  "consolidator": { ... },
  "affection": { ... }
}
```

`aggressiveness` 取值：`"conservative"` / `"standard"` / `"aggressive"`

各插件 `on_tick` 逻辑：
1. 先读 `autopilot.enabled`
2. 若为 true → 忽略分项开关，按 aggressiveness 级别执行全自动流程
3. 若为 false → 读各自分项开关，行为不变

**各级别对应行为：**

| 行为 | conservative | standard | aggressive |
|------|-------------|----------|-----------|
| 自动提取 | ✓ | ✓ | ✓ |
| AI 审核 | ✗（留待审） | ✓ | ✓ |
| 自动通过阈值 | — | 置信度 ≥0.8 | 置信度 ≥0.6 |
| 低置信度处理 | 留候选 | 留待审 | 通过 |
| 漂移自动归档 | ✗ | 超30天归档 | 超14天归档 |
| 表达提取 | ✓ | ✓ | ✓ |
| 记忆整合 | ✓ | ✓ | ✓ |
| 亲密度计分 | ✓ | ✓ | ✓ |

## 技术选型

- 趋势图：复用 `SparklineChart.vue`（KPI 行），大图用 naive-ui n-statistic + SVG path
- 列表行：CSS Grid 固定列宽，hover 高亮，行内按钮
- 设置表单：naive-ui `n-form` + `n-input-number` / `n-switch` / `n-time-picker`
- 状态管理：单个 composable (`useLearningConsole`) 管理三视图数据，按需 lazy fetch
- 时间聚合：前端按 dayjs 分组（today/yesterday/this-week/older）

---

## 与现有系统的关系

- 旧 `LearningView.vue` 保留在磁盘，路由指向新组件
- 现有 slot 系统暂不复用——新页面按 stage 组织，不按 noun 分 slot
- 详情展开内如需 noun-specific 操作，按需 import 对应 composable
- 所有现有 API 保持不变，仅新增 2 个只读 API

## 设计决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 阶段内布局 | 时间聚合行列表 | 用户反馈旧页面词条零散；按时间聚合自然成组 |
| 详情展示 | 行内展开 | 比抽屉轻量，不打断浏览流 |
| 三视图结构 | Tab 切换 | 仪表盘/工作流/设置是不同使用场景 |
| 不复用 slot | 新建组件 | 旧 slot 按 noun 分治，新设计按 stage 分治 |
| 不区分 noun 标签 | 混合展示 | 用户明确要求 |
