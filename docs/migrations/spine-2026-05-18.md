**契约重谈方案**：选定方案 A（在 LLMRequest 加 `auto_record_usage` 开关）。原因：
- 比方案 B（接受多行 usage）侵入面小，旧 admin/usage 视图不用改
- 比方案 C（spine 内部聚合 context manager）实现简单，无需引入会话状态
- 调用方只需一处 `auto_record_usage=False` 就能保留聚合语义
- spine 仍负责 `_record_cache_diagnostic` per-round，break 轴可见性零损失

## 上线后的 task 名（usage.db 应出现）

spine 上线后，`usage.db` 的 `call_type` 列应出现以下新值（按 C.1-C.8 顺序）：

4. `memo` — 记忆卡片提取
5. `slang` — 黑话候选提取
6. `slang_review` — 黑话候选复核
7. `slang_drift` — 黑话语义漂移判定
8. `slang_semantic` — 黑话三阶段语义复核
9. `thinker` — 思考决策（已存在，但 P0-A 后 cache prefix 才稳定）
